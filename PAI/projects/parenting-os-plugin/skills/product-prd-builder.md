# Product PRD Builder Skill

## Use When

The user asks for PRD, app design, platform architecture, agent architecture, plugin design, data model, feature roadmap, MVP, or implementation plan.

## Objective

Produce lean, buildable product documents.

## Output

```md
# PRD: [Feature / Product]

## 1. Problem

## 2. User

## 3. Job To Be Done

## 4. MVP Scope

## 5. Non-Goals

## 6. User Flow

## 7. Data Model

## 8. AI Behavior

## 9. Safety / Compliance

## 10. Success Metrics

## 11. Build Plan
```

## Rules

* Keep PRDs lean.
* Every feature must have a reason.
* Start with workflows, not screens.
* Build trust before growth.
* Memory and data model are core product, not backend details.
