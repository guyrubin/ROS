Arbor is an **AI operating system for child development** — longitudinal milestone tracking (CDC/AAP-2022), AI parenting coaching, child speech-language practice, and a developmental memory that compounds per child. The fundable **innovation core** (the part that is real R&D, not a CRUD app) is a **multimodal, longitudinal developmental-assessment engine**: it turns everyday parent-captured signals (speech, photos, observations, milestones) into a clinically-grounded developmental trajectory + personalised intervention. That engine — not the shipped app — is what every R&D grant is funding.

Funder framings (pick per program): **digital health / preventive child health** (IIA health, Health~Holland, EIT Health), **EdTech** (BIRD, GESAwards, Eurostars), **AI/deep-tech** (EIC — contingent, see §4).

## 2. The structuring play — how a multinational stacks the most (legally)

The instruments **self-separate by geography of activity**, which is exactly what the no-double-funding rules require. Run two R&D streams:

We **already have a Netherlands entity** (the HollandVest vehicle — confirm exact legal name/form) and the intent is to **expand to further European entities** (BE/Flanders next, then DE/FR per the locale matrix). That turns the plan from "stand up one EU company" into a **multi-entity European footprint where each country entity unlocks that country's national R&D scheme** — the more legitimate operating entities, the more parallel non-dilutive pots (no double-funding, because each funds *its own* country's R&D activity).

| Stream | Entity | Activity (must be distinct work-packages) | Claims |
| :-- | :-- | :-- | :-- |
| **Israel R&D** | **Claude Consulting** (IL; owner Karen Rubin) | The developmental-assessment engine + Israeli engineering | **IIA** (Tnufa now → R&D Fund later), **BIRD** (with a US partner) |
| **NL R&D** | **existing NL entity** (HollandVest BV?) | A *separately-scoped* NL stream (e.g. clinical-validation / EU data engine) with **NL payroll** | **WBSO**, **Eurostars** (NL-funded partner), **Health~Holland**, **MIT** |
| **BE R&D** (next) | **new Flemish company** | A distinct Flanders stream (e.g. localisation / EU go-to-market R&D) | **VLAIO Ontwikkelingsproject**, **EIT Health Headstart**, imec.istart |
| **DE/FR R&D** (later) | per-market entities as expansion proceeds | Market-specific R&D | national schemes (e.g. BPIFrance/FR, ZIM/DE) — research per entity |

> ⚠️ **Activity-fit caveat (same trap as Claude Consulting):** the existing NL entity is a **real-estate** vehicle. A Dutch R&D subsidy (WBSO/Eurostars) requires the claiming entity to actually *perform software R&D with NL payroll*. Options: (a) add a documented R&D activity + R&D staff to the existing BV, or (b) spin a **sister/subsidiary BV** for Arbor R&D under the same NL footprint. The NL advisor (D5) picks the cleaner route — but the foothold (Dutch establishment, banking, KvK registration) is already a real head-start.

**Three rules that keep this legal and clawback-free:**
1. **IIA-funded IP stays in Israel.** Transferring know-how out of the Israeli grantee triggers IIA consent + a buy-out of **~3×–6× the grant**. Use arm's-length **IP-licence / R&D-services agreements** between entities — never a transfer. → Decide Arbor's IP home *now* (§7-D2).
2. **No cost funded twice.** Separate cost centres, separate work-package scopes, clean timesheets (each hour claimed once), respect **GBER** aid-intensity ceilings on any shared cost. "Different activity" must be provable on paper.
3. **Eurostars is the cross-border spine** — built for exactly this: an IL + EU consortium where **the EU entity is the funded party** (Israel has €0 allocated Eurostars budget in 2026, so the IL side self-funds or uses IIA).
