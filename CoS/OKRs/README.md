# ROS OKRs — quarterly objectives per department

**Owner:** CoS conductor (COO) · **Cadence:** set/reviewed quarterly (`okr-tracker` skill) · part of [ROS-STRATEGY.md](../ROS-STRATEGY.md) Theme M (the management layer).

Each ROS "department" (mesh) carries 1–3 objectives with measurable key results, reviewed Green/Amber/Red in the weekly review and reset quarterly. OKRs make the company **directed**, not just busy. One file per quarter: `OKRs/Q<n>-<year>.md`.

## How it runs
1. **Set (quarter start):** the conductor drafts OKRs per department from `ROS-STRATEGY.md` + each domain's `MEMORY.md` `Next:`; Guy approves.
2. **Track (weekly):** each KR gets a G/A/R + a number; surfaced in the weekly review + the cockpit.
3. **Close (quarter end):** score, learn, reset.

## Template (`OKRs/Q<n>-<year>.md`)
```md
# OKRs — Q<n> <year>
Reviewed: <date>

## CoS — run the company
- O: <objective>
  - KR1: <measurable> — [G/A/R, current]
## HV — the ultimate EU RE developer
- O: ...
## PAI — product (Arbor)
- O: ...
## KK — the ultimate PA / second brain
- O: ...
## EA · MKT · FIN · Career
- O: ...
```
