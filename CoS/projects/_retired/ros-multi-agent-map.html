Safety & maturity

Governance model

Two ladders govern the system: safety levels gate action by reversibility, and maturity levels track how hardened each capability is. Level 0–2 auto-execute; Level 3+ requires confirmation.

Safety levels — the execution gate

0
Analyze / summarize / researchAuto-execute · no confirmation
1
Draft content (email, docs, plans)Auto-execute · no confirmation
2
Write / edit files in workspaceAuto-execute · no confirmation
3
External action (email, post, API)Require confirmation
4
Financial action (payment, contract)Confirm + state amounts
5
Irreversible (delete, legal, transfer)Confirm with explicit warning

Capability maturity — how hardened

0
Ad hocOne-off behavior, no durable pattern
1
CatalogedNamed in a capabilities/domain file
2
RepeatableHas a checklist / template / command
3
Gated ← default targetSafety, source, freshness, verification checks
4
MeasuredRecords quality, time, cost, failure signals
5
DelegatedSafe reversible work runs by automation; risk gates to Guy
