# Operations Room — Rubin OS CEO cockpit

**Register:** product (design serves the task; Linear/Vercel/Raycast bar)

## What it is
A single self-contained `index.html` control surface for Guy (CEO of a multi-company group). It reads `state.json` (generated by `gen-state.mjs` from the real ROS markdown files) with a baked-in fallback, and renders the running system on one screen: portfolio health, the decisions on Guy's desk, the autonomous loops firing on a clock, the connectors triage board, and every backlog queue.

## Who uses it, where
Guy, at a desk, dark room or bright office, in a triage mindset: "what needs me right now, and is the machine actually running?" The instrument-panel metaphor is literal: this is a cockpit, not a marketing page.

## Truth is the product
The dashboard must never lie. Counts come from the real files; struck/RETRACTED backlog ids are excluded; freshness is stamped and shown. A computed number beats a hand-curated one every time.

## Design system (dark-first instrument panel)
- Palette: OKLCH, deep green-tinted base, emerald as the one structural accent (= live/active/focus). Per-company hue is a data dimension (badge + dot only). Status semantics fixed (green/amber/red/held).
- Type: Inter (UI), Frank Ruhl Libre (serif, view H1 + company names only), mono (all ids/crons/counts/timestamps). Fixed rem scale.
- Motion: strict 150-250ms, state-not-decoration, full prefers-reduced-motion kill-switch. One ambient pulse (live dot), one ticking now-marker.
- Density via tables and row-lists. Cards only for the 4 portfolio tiles.

## Bans (enforced)
No side-stripe accents, no gradient text, no glassmorphism default, no hero-metric block, no per-section uppercase eyebrow, no 01/02/03 scaffolding, no gradient decision card, no body radial-gradient wash, no em dashes in UI copy. Sentence case.
